# Лабораторная работа №1. Wireshark: Введение

## Основное задание

1. Анализатор пакетов Wireshark установлен
2. Wireshark запущен. В главном окне отображаются все доступные интерфейсы
   ![Рис. 1](attachments/Pasted%20image%2020260106010939.png)
3. Выберем интерфейс WSL, запустим захват и в браузере NetSurf перейдем по адресу <http://www.fsb.ru/> (выбран сайт, работающий на HTTP). Остановим захват.
4. Отфильтровуем результаты по протоколу HTTP и найдём пример GET-запроса. В HTTP-заголовке запроса видно URL сайта, а также название браузера.
   ![Рис. 2](attachments/Pasted%20image%2020260106031144.png)

## Дополнительные задания

1. Среди захваченных пакетов имеются примеры работы таких протоколов, как HTTP/1.1, TCP, DNS. Если установить привязку по времени к GET-запросу по запрошенному URL, будет видно, что ответ "OK" с типом text/html (код страницы) получен спустя 0.043163 с.
   ![Рис. 3](attachments/Pasted%20image%2020260106031050.png)
2. IP-адрес, соответствующий домену <www.fsb.ru>: 213.24.76.23. Мой IP-адрес (частный): 172.30.226.180.
3. Распечатать сообщения GET и OK через Wireshark не удалось (приложение не реагирует на нажатие кнопки "Печатать..."), поэтому в PDF-файле представлены скриншоты: ![lab1_GET_OK](attachments/lab1_GET_OK.pdf)
